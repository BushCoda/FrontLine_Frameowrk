# 🤝 Contributing
### FrontLine Framework — Arma Reforger Conflict PvE

---

## Overview

Thank you for your interest in contributing to FrontLine Framework. This document outlines the guidelines for contributing to the project — whether that is reporting bugs, suggesting features, or submitting code changes.

---

## 📋 Before You Contribute

- Read the [README.md](./README.md) to understand the scope of the project.
- Read the [ROADMAP.md](./ROADMAP.md) to understand what is currently being worked on.
- Check the [Issues](https://github.com/BushCoda/FrontLine_Frameowrk/issues) tab to see if your bug or suggestion has already been raised.

---

## 🐛 Reporting Bugs

When reporting a bug please include:

1. A clear and descriptive title.
2. Steps to reproduce the issue.
3. Expected behaviour vs actual behaviour.
4. Any relevant screenshots or logs.
5. The current phase/version of the mod you are testing on.

Open a bug report via the [Issues](https://github.com/BushCoda/FrontLine_Frameowrk/issues) tab and apply the `bug` label.

---

## 💡 Suggesting Features

Feature suggestions are welcome but please note:

- The project must reach a **stable v1.0 foundation** before new features are integrated.
- Check the [ROADMAP.md](./ROADMAP.md) to confirm your suggestion is not already planned.
- Open a suggestion via the [Issues](https://github.com/BushCoda/FrontLine_Frameowrk/issues) tab and apply the `enhancement` label.

---

## 🔀 Submitting Changes

If you wish to contribute code or documentation changes:

1. **Fork** the repository.
2. Create a **new branch** from `main` using a descriptive name:
   ```
   feature/your-feature-name
   fix/your-bug-fix
   docs/your-doc-update
   ```
3. Make your changes, keeping commits clear and descriptive.
4. Open a **Pull Request** against the `main` branch.
5. Describe your changes clearly in the PR description.

---

## 📝 Commit Message Format

Please follow this format for commit messages:

```
type: short description

Optional longer description if needed.
```

| Type | Usage |
|---|---|
| `feat` | A new feature |
| `fix` | A bug fix |
| `docs` | Documentation changes |
| `balance` | Gameplay value tuning |
| `refactor` | Code restructure without behaviour change |
| `test` | Adding or updating tests |

---

## ⚠️ Important Notes

- This mod is built exclusively for **Arma Reforger** using **EnforceScript**.
- All scripting must be compatible with the base **Everon** map.
- Do not submit changes that introduce external mod dependencies without prior discussion.
